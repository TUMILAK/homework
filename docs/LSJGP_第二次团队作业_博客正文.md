## 1. 原型设计（40%）

### 1.1 工具与访问

- **原型工具**：墨刀 / Axure（二选一；**请在下方「占位」替换为已开启「公开访问」的在线原型链接**）。
- **原型在线预览链接**：`[https://modao.cc/app/xxxx — 请替换]`

### 1.2 前台应用原型（用户侧）

围绕 **「智能体工作台」** 与 **「阅读 / 爬取 / 设置」** 场景设计，主要包括：

1. **工作台（agent）**  
   - EPUB 上传与汉化进度提示；图片汉化嵌字上传与结果下载链接。  
   - WebSocket 连接、模式选择（ReAct / Plan-and-Execute）、消息输入与事件流日志区。  

2. **设置（settings）**  
   - DeepSeek API Key、Base URL、模型名；后端 Origin；主题与壁纸等偏好。  

3. **爬取（crawl）、书库（library）、阅读器（reader）、首页（index）**  
   - 与抓取结果、本地 workspace 目录联动的导航与占位说明（与实现一致处高保真，未实现处用文字规则说明）。

### 1.3 后台管理原型（管理侧）

当前版本以 **单用户 / 本机或内网部署** 为主，后台侧重 **运维与配置**（非多租户运营后台），原型中体现：

- 服务健康检查入口（`/api/health`）、工作区目录树浏览（`/api/catalog`）的等价管理视图（可为简化页或说明卡片）。  
- 环境变量说明：工作区根路径、MCP 子进程、CORS、端口等（与《系统设计说明书》配置章节一致）。

### 1.4 交互、文本与规则说明（必要）

- **交互**：连接 → 配置密钥 → 发送任务 → 流式事件（phase / tool_call / tool_result）→ 回合结束。  
- **规则**：路径均限制在 `workspace/` 下；下载仅允许相对工作区安全解析；大文件大小上限在界面侧提示。  
- **UI**：深色主题与 Pico 风格统一；关键操作用主按钮与禁用态区分。

### 1.5 原型突出特点（结合文字）

- **双通道**：HTTP 批量任务（EPUB / 图片）与 WebSocket 长连接智能体并存，原型中分区展示，避免用户混淆。  
- **安全提示**：密钥仅存浏览器 LocalStorage 时的风险提示与「可用环境变量」说明。

## 2. 概要设计（40%）

### 2.1 架构与模块（自顶向下）

见团队仓库 `docs/LSJGP_系统设计说明书.md` 中「系统总体结构」「逻辑分层」与 Mermaid 图；此处摘要如下：

- **表现层**：静态 Web（`frontend/web`），通过 `fetch` / `WebSocket` 调用后端。  
- **应用层**：FastAPI 路由、鉴权与参数校验、WebSocket 会话编排。  
- **领域层**：Agent 引擎（ReAct / Plan-Execute）、EPUB 汉化、图片 OCR+嵌字。  
- **基础设施层**：MCP stdio 子进程、工作区文件系统、HTTP 客户端（httpx / OpenAI SDK）。

### 2.3 数据库设计摘要

- **当前实现**：持久化以 **工作区目录文件** 为主，浏览器 **LocalStorage** 存用户偏好；**无中心关系型库**。  
- **设计说明书** 中给出 **逻辑 ER**、**规范化关系模型（规划表）** 与 **未来 ORM 映射**，便于二期扩展用户与作业队列。

详见 `docs/LSJGP_数据库设计说明书.md`。
